# Review Checklist

Use this checklist before completing a change.

- The current task in `.ai/current-priority.md` was respected.
- Non-negotiable rules in `.ai/coding-rules.md` were followed.
- Relevant `spec/` documents were consulted for the changed subsystem.
- Only intended files were modified.
- Required tests or validation commands were run.
- New behavior aligns with existing contracts, schemas, and boundaries.
