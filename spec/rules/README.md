# Rule Engine

## Purpose

`spec/rules/` defines the future deterministic Rule Engine boundary.

The Rule Engine transforms authoritative requirements and approved policy into executable deterministic rules and traceable evaluation results.

## Current boundary

The Rule Engine is not yet implemented.

It will remain distinct from:

- source knowledge
- AI recommendations
- report prose generation

## Core principles

- requirements are knowledge
- rules are executable deterministic logic
- rules must cite their authoritative sources
- rule execution must remain possible without AI
- evaluation results must be auditable

See `spec/rules/rule-engine-boundary.md` for the boundary definition.
